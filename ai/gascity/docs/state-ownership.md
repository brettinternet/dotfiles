# Gas City state ownership

This is the state contract for the installed Gas City 1.3.5, Beads 1.1.0,
Dolt 2.2.0, and the sidecar. A component may observe or cache another
component's state without becoming its owner. In particular, the sidecar
SQLite database is never execution truth.

## Boundaries

- **Authority does not flow through caches.** Gas City events and sidecar views can
  report what was observed, but Beads owns task and workflow execution state. A
  sidecar restart or a missing event cannot by itself alter a Beads outcome.
- **Event retention is bounded.** The sidecar keeps a bounded recent mapped view
  (currently 100 events) and persists the latest processed sequence. Replay is
  requested after that cursor. A rotated Gas City log may have no marker that
  relates its new head to the old log, so events discarded before rotation are
  unrecoverable by the sidecar.
- **Secrets and paths stay local.** Sidecar settings use `GC_SIDECAR_` environment
  variables and may read a local `.env`; Pushover delivery requires
  `PUSHOVER_APP_TOKEN` and `PUSHOVER_USER_KEY`. `.env`, credentials, absolute
  rig paths, and machine bindings must not enter tracked Git or generated
  portable policy.
- **Writeback is a separate capability.** Reading, previewing, importing, and
  materializing an external backlog item must leave its source bytes unchanged.
  Only an explicit guarded writeback operation may update that source; the
  sidecar database, event stream, and Beads state do not grant that permission.

## Recovery order

1. Restore the tracked Git tree and install the versions recorded in
   `docs/environment.md`.
2. Recreate local `.gc/` bindings and Gas City runtime state; provision secrets
   through environment/secret inputs rather than Git.
3. Reconnect or restore the rig's Dolt-backed Beads store and inspect its
   existing tasks and workflow records before resuming work.
4. Restore the rig artifacts under `.gascity/work/<root>/` and reconcile them
   with Beads; missing evidence is not an execution outcome.
5. Start the city, then start the sidecar. Restore its SQLite state when
   available. If it is absent, establish desired policy deliberately, expect
   replay only from the event log's available head, and expect notification
   dedupe/pending-delivery state to be rebuilt or retried.
6. Re-read and preview the external backlog. Re-materialize execution copies
   without writing the source; use guarded explicit writeback only when the
   operator deliberately authorizes the source update.
